Games

Different games, same shared universe underneath — the same metaverse thread FIELDOFFICE/TrapX's alive city runs through, not each game's literal setting. One server-authoritative engine lineage, built in the open.

FPS · live development

SHANKPIT

A fast, stylized, server-authoritative UDP FPS being built into the front end of a persistent world engine (DragonsNShit / Dragonfly). Multi-scene portals, neural-net bots, team modes, and — newest — Bedrock Racers, an F1-tier vehicle racing mode with an item/ultimate layer on top.

github.com/emilyspringerton/SHANKPIT
FPS · esports fork

shankpit-460

Forked from SHANKPIT at tag 460 with a distinct mission: a stripped-down, low-system-spec competitive esports FPS, tuned purely for competitive performance so it reaches players on modest hardware. Does not carry forward the persistent-world ambitions — those stay in the parent repo.

github.com/emilyspringerton/shankpit-460
Platform fighter

BRAWLPIT

A 2.5D platform fighter built on the SHANKPIT engine (Build 178) — momentum-driven combat, shields, wavedashing, and an up-B recovery, in the same spirit as the genre's classics but running on our own server-authoritative UDP stack.

Card-deck RTS / MOBA · early access waitlist open

RED GARDEN

Draw cards, deploy autonomous units onto a living cellular-automata board, and watch systems collide — Clash Royale's card economy over a board that's alive on its own tick. Also iterating toward a mouse-driven MOBA mode with a named hero roster.
